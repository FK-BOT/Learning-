1. Basic Syntax & Data Types

  • Variables — no type declaration needed: x = 5, name = "farhan"
  • Types: int, float, str, bool, None
  • Type checking: type(x) → returns <class 'int'>
  • Type conversion: int("5"), str(42), float("3.14")
  • String methods: .upper(), .lower(), .strip(), .split(), .replace(), .find()
  • f-strings: f"Hello {name}" — modern way to format
# Variables and types age = 22 name = "Farhan" gpa = 7.5 is_placed = False # f-string print(f"Name: {name}, Age: {age}") # String methods s = " hello world " print(s.strip()) # "hello world" print(s.upper()) # " HELLO WORLD " print(s.split()) # ['hello', 'world']

2. Collections — List, Tuple, Dict, Set

  • List — ordered, mutable: [1, 2, 3]
  • Tuple — ordered, immutable: (1, 2, 3)
  • Dict — key-value pairs: {"name": "Farhan"}
  • Set — unordered, unique values: {1, 2, 3}
  • List slicing: lst[1:3], lst[::-1] (reverse)
  • Dict access: d["key"] or d.get("key", default)
fruits = ["apple", "banana", "cherry"] fruits.append("mango") fruits.remove("banana") print(fruits[0]) # apple print(fruits[-1]) # mango person = {"name": "Farhan", "age": 22} person["city"] = "Bangalore" print(person.keys()) # dict_keys(['name','age','city']) print(person.values()) # dict_values(['Farhan', 22, 'Bangalore']) # Loop through dict for k, v in person.items(): print(k, "→", v)

3. Conditionals & Loops

  • if / elif / else — standard branching
  • for loop — iterate over list, range, dict
  • while loop — runs while condition is true
  • break — exit loop; continue — skip iteration
  • range(start, stop, step) — e.g. range(0, 10, 2)
  • List comprehension: [x*2 for x in lst if x > 0]
# For loop with range for i in range(1, 6): print(i) # 1 2 3 4 5 # While loop count = 0 while count < 3: print("tick") count += 1 # List comprehension squares = [x**2 for x in range(1, 6)] # [1, 4, 9, 16, 25]

4. Functions

  • Define with def, call by name
  • Default arguments: def greet(name="World")
  • *args — variable positional args (tuple)
  • **kwargs — variable keyword args (dict)
  • return — send value back; functions return None by default
  • Lambda: square = lambda x: x**2
def add(a, b=0): return a + b print(add(3, 5)) # 8 print(add(10)) # 10 (uses default b=0) # *args example def total(*nums): return sum(nums) print(total(1, 2, 3, 4)) # 10 # Lambda square = lambda x: x**2 print(square(5)) # 25

5. File I/O

  • open(file, mode) — modes: 'r' read, 'w' write, 'a' append
  • Always use with open(...) as f: — auto-closes file
  • f.read() — full content; f.readlines() — list of lines
  • f.write("text") — write string
# Writing to file with open("test.txt", "w") as f: f.write("Hello Mavenir\n") f.write("Line 2\n") # Reading from file with open("test.txt", "r") as f: for line in f: print(line.strip())

6. Exception Handling

  • try / except / finally — handle runtime errors
  • Common exceptions: ValueError, TypeError, FileNotFoundError, ZeroDivisionError, IndexError, KeyError
  • finally block always runs (cleanup)
  • raise Exception("msg") — throw custom error
try: result = 10 / 0 except ZeroDivisionError as e: print(f"Error: {e}") finally: print("This always runs") # Multiple exceptions try: val = int("abc") except ValueError: print("Not a number")
💡 Key insight for the test: Python uses indentation (4 spaces) instead of curly braces. No semicolons. Variables don't need type declarations. These are the most common "gotchas" in written tests.

1. Navigation & File System

  • pwd — print current directory
  • ls — list files; ls -la — long format with hidden
  • cd /path — change directory; cd ~ home; cd .. up
  • mkdir dirname — create directory
  • touch file.txt — create empty file
  • cp src dest — copy; mv src dest — move/rename
  • rm file — delete file; rm -rf dir — delete dir recursively
  • cat file — print file; less file — paginated view

2. File Permissions

  • Format: -rwxrwxrwx → owner / group / others
  • r=4, w=2, x=1 → rwx = 7, rw- = 6, r-- = 4
  • chmod 755 file → owner: rwx, group: r-x, others: r-x
  • chmod 644 file → owner: rw-, group: r--, others: r--
  • chown user:group file — change ownership
  • chmod +x script.sh — make executable
# View permissions ls -l myfile.py # -rw-r--r-- 1 farhan farhan 245 May 8 09:00 myfile.py # Give owner full, others read only chmod 744 myfile.py # Make script executable chmod +x run.sh ./run.sh

3. Processes

  • ps aux — list all running processes
  • top / htop — live process monitor
  • kill PID — terminate process; kill -9 PID force kill
  • ps aux | grep python — find specific process
  • & — run in background: ./script.sh &
  • nohup command & — run even after logout
  • jobs — list background jobs; fg — bring to foreground

4. grep, awk, sed — Text Processing

  • grep "pattern" file — search for pattern
  • grep -i case-insensitive; grep -r recursive; grep -n line numbers
  • grep -v "pattern" — exclude lines matching pattern
  • awk '{print $1}' file — print 1st column
  • awk -F":" '{print $1}' /etc/passwd — use : as delimiter
  • sed 's/old/new/g' file — replace all occurrences
  • sed -i 's/old/new/g' file — edit in-place
# grep examples grep "error" app.log # find "error" in log grep -i "ERROR" app.log # case-insensitive grep -rn "def main" ./src/ # recursive with line nums # awk — print 2nd column of CSV awk -F"," '{print $2}' data.csv # sed — replace in file sed 's/localhost/0.0.0.0/g' config.txt

5. Pipes, Redirection & Useful Commands

  • | pipe — pass output to next command
  • > redirect output to file (overwrites); >> append
  • wc -l file — count lines
  • sort file — sort lines; sort -r reverse
  • uniq — remove duplicate adjacent lines
  • head -n 10 file / tail -n 10 file — first/last 10 lines
  • find /path -name "*.log" — find files by name
  • df -h — disk space; du -sh dir — dir size
  • tar -czf archive.tar.gz dir/ — create tar.gz
# Chaining commands ps aux | grep python | grep -v grep cat access.log | grep "404" | wc -l ls -la | sort -k5 -n | tail -5 # Redirect output ./test.sh > output.txt 2>&1 echo "new line" >> notes.txt

6. Shell Scripting Basics

  • First line: #!/bin/bash (shebang)
  • Variables: NAME="Farhan", use as $NAME
  • Read input: read varname
  • Conditionals: if [ condition ]; then ... fi
  • File tests: -f file exists, -d dir is directory
  • For loop: for i in 1 2 3; do ... done
#!/bin/bash NAME="Farhan" echo "Hello $NAME" # Check if file exists if [ -f "test.txt" ]; then echo "File exists" else echo "File not found" fi # Loop for i in 1 2 3; do echo "Count: $i" done

1. The 4 Pillars of OOP

  • Encapsulation — bundle data + methods inside a class; hide internals
  • Abstraction — expose only what's needed; hide implementation details
  • Inheritance — child class gets properties of parent class
  • Polymorphism — same method name, different behaviour per class

2. Classes & Objects in Python

  • class ClassName: — define a class
  • __init__(self, ...) — constructor, called when object is created
  • self — refers to the current instance (like Java's this)
  • Instance variables: self.name = name
  • Create object: obj = ClassName(args)
class Animal: def __init__(self, name, species): self.name = name self.species = species def speak(self): print(f"{self.name} makes a sound") dog = Animal("Rex", "Dog") dog.speak() # Rex makes a sound print(dog.name) # Rex

3. Inheritance & Polymorphism

  • Child class: class Dog(Animal):
  • super().__init__() — call parent constructor
  • Method overriding — redefine parent method in child
  • Polymorphism — call same method on different objects, get different results
class Dog(Animal): def __init__(self, name): super().__init__(name, "Dog") def speak(self): # overrides parent print(f"{self.name} says Woof!") class Cat(Animal): def speak(self): print(f"{self.name} says Meow!") animals = [Dog("Rex"), Cat("Luna")] for a in animals: a.speak() # Polymorphism in action

4. Encapsulation — Private Variables

  • _var — single underscore: convention for "protected" (don't use outside)
  • __var — double underscore: name-mangled, truly private
  • Getter/setter methods control access to private data
class BankAccount: def __init__(self, balance): self.__balance = balance # private def get_balance(self): return self.__balance def deposit(self, amount): if amount > 0: self.__balance += amount acc = BankAccount(1000) acc.deposit(500) print(acc.get_balance()) # 1500
💡 You know OOP from Java — the concepts are identical. Just remember Python syntax differences: self instead of this, def __init__ instead of constructor name, and no access modifiers (public/private/protected keywords).
🗄 SQL & Database Fundamentals PREFERRED
~1.5 hrs ▾

1. Basic SELECT Queries

  • SELECT * FROM table — all columns
  • SELECT col1, col2 FROM table WHERE condition
  • ORDER BY col ASC/DESC — sort results
  • LIMIT n — restrict result count
  • DISTINCT — remove duplicate rows
  • Operators: =, !=, >, <, LIKE, IN, BETWEEN, IS NULL
-- Basic queries SELECT * FROM employees; SELECT name, salary FROM employees WHERE salary > 50000; SELECT DISTINCT department FROM employees; SELECT * FROM employees ORDER BY salary DESC LIMIT 5; -- LIKE, IN, BETWEEN SELECT * FROM employees WHERE name LIKE 'F%'; SELECT * FROM employees WHERE dept IN ('IT', 'QA'); SELECT * FROM employees WHERE salary BETWEEN 40000 AND 70000;

2. Aggregate Functions & GROUP BY

  • COUNT(*) — total rows; COUNT(col) — non-null values
  • SUM(col), AVG(col), MAX(col), MIN(col)
  • GROUP BY col — group rows with same value
  • HAVING condition — filter after GROUP BY (like WHERE but for groups)
-- Count employees per department SELECT department, COUNT(*) AS total FROM employees GROUP BY department; -- Average salary per dept, only dept with avg > 60k SELECT department, AVG(salary) AS avg_sal FROM employees GROUP BY department HAVING AVG(salary) > 60000;

3. JOINs

  • INNER JOIN — only rows that match in BOTH tables
  • LEFT JOIN — all rows from left table + matching from right (NULL if no match)
  • RIGHT JOIN — all rows from right table + matching from left
  • FULL OUTER JOIN — all rows from both tables
-- Tables: employees(id, name, dept_id), departments(id, dept_name) -- INNER JOIN — only matched rows SELECT e.name, d.dept_name FROM employees e INNER JOIN departments d ON e.dept_id = d.id; -- LEFT JOIN — all employees, even without dept SELECT e.name, d.dept_name FROM employees e LEFT JOIN departments d ON e.dept_id = d.id;

4. DDL & DML — CREATE, INSERT, UPDATE, DELETE

  • DDL (Data Definition): CREATE, ALTER, DROP, TRUNCATE
  • DML (Data Manipulation): SELECT, INSERT, UPDATE, DELETE
  • Primary Key — unique identifier for a row; cannot be NULL
  • Foreign Key — links to primary key of another table
CREATE TABLE employees ( id INT PRIMARY KEY, name VARCHAR(100) NOT NULL, salary DECIMAL(10,2), dept_id INT, FOREIGN KEY (dept_id) REFERENCES departments(id) ); INSERT INTO employees VALUES (1, 'Farhan', 55000, 2); UPDATE employees SET salary = 60000 WHERE id = 1; DELETE FROM employees WHERE id = 1;

5. Normalization Basics

  • 1NF — no repeating groups; each column has atomic values
  • 2NF — 1NF + no partial dependency (non-key cols depend on full PK)
  • 3NF — 2NF + no transitive dependency (non-key cols depend only on PK)
  • Index — speeds up SELECT queries; slows INSERT/UPDATE
  • CREATE INDEX idx_name ON table(column)

1. OSI Model — 7 Layers

  • 7 – Application — HTTP, FTP, SMTP, DNS (user-facing protocols)
  • 6 – Presentation — encryption, encoding, compression (SSL/TLS)
  • 5 – Session — manages sessions/connections between apps
  • 4 – Transport — TCP (reliable), UDP (fast, unreliable), ports
  • 3 – Network — IP addresses, routing (IP, ICMP)
  • 2 – Data Link — MAC addresses, switches, frames (Ethernet)
  • 1 – Physical — cables, signals, hardware (bits)
Mnemonic: "All People Seem To Need Data Processing" (top to bottom: App→Physical)

2. TCP vs UDP

  • TCP — connection-oriented, reliable, ordered, slower. Uses 3-way handshake (SYN → SYN-ACK → ACK)
  • UDP — connectionless, faster, no guarantee of delivery. Used for streaming, DNS, gaming
  • TCP examples: HTTP, HTTPS, FTP, SSH, SMTP
  • UDP examples: DNS, VoIP, video streaming, DHCP

3. Important Ports

  • HTTP → 80 | HTTPS → 443
  • FTP → 20/21 | SSH → 22 | Telnet → 23
  • SMTP → 25 | DNS → 53 | DHCP → 67/68
  • POP3 → 110 | IMAP → 143 | MySQL → 3306
  • PostgreSQL → 5432 | Redis → 6379 | MongoDB → 27017

4. IP Addressing & Subnetting Basics

  • IPv4 — 32-bit: e.g. 192.168.1.1 (4 octets, 0–255 each)
  • IPv6 — 128-bit: e.g. 2001:0db8::1 (8 groups of hex)
  • Private ranges: 10.x.x.x, 172.16.x.x, 192.168.x.x
  • Loopback: 127.0.0.1 (localhost)
  • Subnet mask: /24 = 255.255.255.0 → 256 addresses, 254 usable
  • CIDR notation: 192.168.1.0/24

5. HTTP & Web Basics

  • HTTP methods: GET (read), POST (create), PUT (replace), PATCH (partial update), DELETE
  • Status codes: 2xx success, 3xx redirect, 4xx client error, 5xx server error
  • Common: 200 OK, 201 Created, 400 Bad Request, 401 Unauthorized, 403 Forbidden, 404 Not Found, 500 Internal Server Error
  • REST API — stateless, uses HTTP methods on resources
  • DNS — translates domain names to IP addresses

6. Telecom Concepts (Mavenir-specific)

  • 4G LTE / 5G — mobile network generations. 5G = sub-6GHz + mmWave, lower latency
  • VoIP — Voice over IP; Mavenir's core domain. Uses SIP protocol
  • SIP (Session Initiation Protocol) — sets up/tears down voice/video calls over IP
  • RTP — Real-time Transport Protocol; carries actual audio/video data
  • IMS — IP Multimedia Subsystem; framework for delivering multimedia over IP networks
  • NFV — Network Function Virtualization; running telecom functions as software (VMs/containers)
  • vRAN / Open RAN — virtualized radio access network; Mavenir's key product area
